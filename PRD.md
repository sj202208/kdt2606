# PRD — 퀴즈 학습 사이트

## 1. 서비스 정의

학생이 로컬 환경에서 이름과 비밀번호로 로그인한 뒤, quiz.csv에서 랜덤 추출된 5문제를 1화면 1문제씩 풀고, 풀이 결과를 JSON 파일로 기록·확인할 수 있는 객관식 퀴즈 학습 사이트.

---

## 2. 타겟 유저

- **사용자**: 학생 (교사가 관리하는 계정 보유)
- **상황**: 로컬 PC 브라우저에서 수업 중 또는 자습 시간에 접속
- **기기**: 데스크톱 / 노트북 (PC 우선)
- **디지털 친숙도**: 낮음 ~ 중간 (단순한 UI 필요)

---

## 3. 사용자 시나리오

### 성공 흐름

1. 학생이 브라우저에서 로컬 주소(`http://localhost:5000`)에 접속한다.
2. 로그인 페이지에서 이름과 비밀번호를 입력하고 **문제풀기** 버튼을 클릭한다.
3. 인증이 성공하면 문제풀기 페이지로 이동한다.
4. quiz.csv에서 랜덤 추출된 5문제가 1화면에 1문제씩 출제된다.
5. 학생이 보기(A~D)를 선택하고 **다음 문제** 버튼을 클릭하면 다음 문제가 표시된다.
6. 5번째(마지막) 문제에서는 버튼이 **결과보기**로 바뀐다.
7. **결과보기** 버튼을 클릭하면 결과가 `results/학생이름.json`에 저장(append)된 뒤 결과보기 페이지로 이동한다.
8. 결과보기 페이지에서 총점, 각 문제별 O/X, 문제·정답·풀이를 확인한다.
9. 하단의 **문제풀기** 버튼을 클릭하면 새로운 5문제가 랜덤 추출된 문제풀기 페이지 처음으로 이동한다.

### 예외 상황

- **로그인 실패**: 이름 또는 비밀번호가 일치하지 않으면 로그인 페이지에 오류 메시지를 표시하고 다시 입력을 요청한다. 페이지 이동 없음.
- **입력값 미입력**: 이름 또는 비밀번호가 빈 칸인 경우 제출을 막고 안내 메시지를 표시한다.
- **보기 미선택 상태에서 다음 이동 시도**: 보기를 선택하지 않고 버튼을 클릭하면 경고 메시지를 표시하고 이동하지 않는다.
- **quiz.csv 문제 수 부족**: csv에 5문제 미만이 존재할 경우, 있는 문제 수만큼만 출제한다.

## 4. 프로젝트 범위

### 반드시 되어야 함 (Scope In)

- 이름 + 비밀번호 입력 후 SQLite3 DB에서 인증
- quiz.csv에서 5문제 랜덤 추출 (세션 내 중복 없음)
- 1화면 1문제 방식의 문제풀기 (A·B·C·D 객관식)
- 문제 진행 표시 (`{번호}/5` 형식)
- 마지막 문제에서 버튼이 "결과보기"로 전환
- 결과를 `results/학생이름.json`에 append 저장 (기존 기록 유지, 덮어쓰기 금지)
- 결과보기 페이지에서 총점, 문제별 O/X, 문제·정답·풀이 전체 표시
- 결과보기 하단의 "문제풀기" 버튼으로 새 세션 시작

### 절대 안 됨 (Scope Out)

- 회원가입 화면 (계정은 관리자가 DB에 직접 추가)
- 문제별 정오 즉시 피드백 (문제풀기 중 정답 표시 금지 — 결과보기에서만 확인)
- 결과 파일 overwrite (기존 기록 삭제 금지)
- 외부 서버 배포 / 클라우드 연동
- 난이도(Difficulty) 필드 사용

## 5. 화면 목록

| # | 화면 이름 | URL (예시) | 화면 역할 |
|---|---|---|---|
| 1 | **로그인** | `/` | 이름·비밀번호 입력 후 인증, 문제풀기로 이동 |
| 2 | **문제풀기** | `/quiz` | 1화면 1문제 출제, 보기 선택, 다음문제/결과보기 이동 |
| 3 | **결과보기** | `/result` | 총점·O/X·문제·정답·풀이 표시, JSON 저장 트리거, 재도전 버튼 |

### 화면 흐름도

```
[로그인 /]
  ├── 인증 성공 → [문제풀기 /quiz] (문제 1~5 반복)
  │                    ├── 1~4번 문제: [다음 문제] → 다음 문제
  │                    └── 5번 문제: [결과보기] → results/이름.json append 저장 → [결과보기 /result]
  │                                                           └── [문제풀기] → [문제풀기 /quiz] (새 세션)
  └── 인증 실패 → 로그인 페이지 오류 메시지 표시
```

## 6. 완료 기준

- [ ] `http://localhost:5000` 접속 시 로그인 페이지가 표시되면 OK
- [ ] 홍길동/1111로 로그인 시 문제풀기 페이지로 이동하면 OK
- [ ] 틀린 비밀번호 입력 시 오류 메시지가 로그인 페이지에 표시되고 이동하지 않으면 OK
- [ ] 문제풀기 페이지에서 매 세션마다 5문제가 랜덤 추출되고 세션 내 중복이 없으면 OK
- [ ] 1화면에 1문제씩 표시되고 헤더에 `{번호}/5`가 표시되면 OK
- [ ] 보기를 선택하지 않고 버튼 클릭 시 경고가 표시되고 이동하지 않으면 OK
- [ ] 5번 문제에서 버튼이 "결과보기"로 바뀌면 OK
- [ ] "결과보기" 클릭 시 `results/학생이름.json` 파일이 생성되고, 재시도 시 기존 기록에 추가(append)되면 OK
- [ ] 결과보기 페이지에서 총점(`OO점 (정답수/5)`), 문제별 O/X, 문제·정답·풀이가 모두 표시되면 OK
- [ ] 결과보기의 "문제풀기" 버튼 클릭 시 새로운 랜덤 5문제 세션이 시작되면 OK

## (부속 A) 운영 제약

- **서비스 환경**: 로컬 전용 (`localhost:5000`), 외부 공개 불필요
- **OS**: Windows (CMD/PowerShell 환경)
- **유지보수**: 비개발자 교사가 quiz.csv 파일과 SQLite3 DB를 직접 수정
- **기기**: PC/노트북 브라우저 우선 (모바일 대응 불필요)
- **인증**: 평문 비밀번호 저장 (로컬 학습용, 보안 강화 불필요)
- **계정 관리**: 회원가입 없음 — 관리자가 DB(`users` 테이블)에 직접 추가, 테스트 계정 `홍길동/1111` 사전 입력

## (부속 B) 데이터 목록

### DB (SQLite3) — `users` 테이블

| 데이터명 | 필드명 | 필수 | 수정 권한 | 민감도 | 저장 위치 |
|---|---|---|---|---|---|
| 학생 이름 | `name` | ✅ | 관리자 | 일반 | DB |
| 비밀번호 | `password` | ✅ | 관리자 | 낮음 | DB |

### CSV — `quiz.csv`

| 데이터명 | 필드명 | 필수 | 설명 |
|---|---|---|---|
| 문제 | `Question` | ✅ | 객관식 문제 본문 |
| 보기 A | `Example_A` | ✅ | 1번 보기 |
| 보기 B | `Example_B` | ✅ | 2번 보기 |
| 보기 C | `Example_C` | ✅ | 3번 보기 |
| 보기 D | `Example_D` | ✅ | 4번 보기 |
| 정답 | `Answer` | ✅ | 정답 보기 (A/B/C/D 중 하나) |
| 풀이 | `Explanation` | ✅ | 결과보기 화면에 표시될 해설 |

### JSON — `results/학생이름.json`

```json
[
  {
    "timestamp": "2026-05-22T07:55:00",
    "score": 3,
    "questions": [
      {
        "question": "문제 본문",
        "answer": "A",
        "student_answer": "B",
        "explanation": "해설 내용",
        "is_correct": false
      }
    ]
  }
]
```

- 파일명: `results/{학생이름}.json` (예: `results/홍길동.json`)
- 신규 세션은 기존 배열에 append, 덮어쓰기 금지

## (부속 C) 예시 입력 / 출력

### 로그인

- 입력: 이름 `홍길동`, 비밀번호 `1111`
- 출력: `/quiz` 페이지로 이동, 5문제 랜덤 추출

### 문제풀기

- 입력: 1번 문제에서 보기 B 선택 후 "다음 문제" 클릭

- 입력: 5문제 모두 풀고 "결과보기" 클릭

```json

  {
    "timestamp": "2026-05-22T08:10:30",

      { "question": "HTML의 기본 구조 태그는?", "answer": "B", "student_answer": "C", "explanation": "<html>, <head>, <body>가 기본 구조입니다.", "is_correct": false }
```
